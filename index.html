🔄 ลำดับการทำงานของระบบ (Workflow)

1. เลือกโฟลเดอร์ Source/Output → 2. สแกนหาไฟล์ NewDocs → 3. อ่านไฟล์ PDF/รูป → 4. ย่อขนาดภาพ → 5. ประมวลผลด้วย Gemini AI → 6. ตรวจสอบเงื่อนไข 4 จุด → 7. เปลี่ยนชื่อ & ย้ายไฟล์

รายละเอียดเจาะลึกแต่ละขั้นตอน

  1. เลือกโฟลเดอร์ — ผู้ใช้เลือก Source Directory (โฟลเดอร์ที่มี NewDocs อยู่ด้านใน) และ Output Directory (โฟลเดอร์ปลายทาง เช่น ProDocs)
  2. ค้นหาไฟล์ — ค้นหาโฟลเดอร์ที่ชื่อขึ้นต้นด้วย NewDocs แล้วดึงไฟล์ *.pdf และรูปภาพ (.jpg .jpeg .png .webp .bmp .tif .tiff .heic)
  3. อ่านไฟล์ — หากเป็น PDF จะอ่านเนื้อหาทั้งไฟล์ หากเป็น รูปภาพ จะใช้ไลบรารี Pillow ในการโหลด
  4. ย่อรูปภาพ (Image Optimization) — รูปภาพจะถูกย่อให้ด้านยาวสูงสุด 2048 px เพื่อประหยัด Token ฝั่ง AI โดยใช้เทคนิค LANCZOS เพื่อรักษาความคมชัด และตั้งค่าบีบอัด JPEG ที่คุณภาพ 85%
  5. ส่งให้ Gemini AI — ส่งเอกสารพร้อม Prompt พิเศษที่บังคับให้ AI ตอบกลับมาเป็นโครงสร้าง JSON เท่านั้น
  6. ตรวจสอบเงื่อนไข — นำผลลัพธ์มาตรวจจุดสำคัญ 4 กรอบแดง พร้อมเช็คความถูกต้องของชื่อและประเภทกู้ หากผิดพลาดแม้แต่จุดเดียว จะส่งไปโฟลเดอร์ Manual
  7. ย้ายไฟล์ — เอกสารที่ผ่านเกณฑ์ จะถูกเปลี่ยนชื่อเป็น ชื่อ_เลขบัตรประชาชน_ประเภทกู้.jpg แล้วย้ายเข้าโฟลเดอร์ที่ตรงกับประเภทการกู้ยืม

🧠 โครงสร้างข้อมูลที่ AI สกัดออกมา (Data Extraction)

ระบบกำหนด Prompt เพื่อให้ Gemini ส่งค่ากลับมาในรูปแบบ JSON ที่มีฟิลด์ดังต่อไปนี้:

{
  "name": "ชื่อ-นามสกุล ผู้กู้",
  "id_card_top": "เลขบัตร 13 หลักบริเวณบาร์โค้ดด้านบน",
  "id_card_body": "เลขบัตร 13 หลักในหัวข้อที่ 1",
  "loan_type": "Type1 | Type2 | Type3 | Type4",
  "signed": true | false
}
⚠️ ข้อควรระวัง: ค่า id_card_top และ id_card_body ถูกกำหนดให้ AI ต้องอ่านแยกกันโดยอิสระ ห้ามดึงค่าจากช่องใดช่องหนึ่งมาเติมให้อีกช่อง เพื่อให้ระบบ Cross-check ได้อย่างแม่นยำ

📦 เงื่อนไข 4 กรอบแดงที่ต้องผ่านการตรวจสอบ

จุดตรวจ (กรอบ) ข้อมูลที่อ่าน เกณฑ์การให้ผ่าน (Pass Criteria)
จุดที่ #1 เลขบัตรประชาชน (บริเวณบาร์โค้ดด้านบน) ต้องเป็นตัวเลขความยาว 13 หลักถ้วน
จุดที่ #2 ข้อความที่ขึ้นต้นด้วย "ลักษณะที่ ..." สามารถแปลงเป็น Type1 – Type4 ได้ถูกต้องตามตัวเลข
จุดที่ #3 เลขบัตรประชาชน (ในพื้นที่หัวข้อที่ 1) ต้องมีค่าตรงกับ จุดที่ #1 ทุกหลัก (Cross-check)
จุดที่ #4 ลายเซ็นผู้กู้ (Signature) พบร่องรอยน้ำหมึก ลายเซ็น หรือจุดใดๆ ถือว่าผ่าน (true)

🖼️ การจัดการรูปภาพ (Image Optimization)

เนื่องจากภาพถ่ายจากกล้องโทรศัพท์มือถือในปัจจุบันมีความละเอียดสูงมาก ระบบจึงต้องทำการปรับลดขนาดก่อนส่งให้ AI เพื่อลดการใช้ Resource และ Token:

  • รูปแบบไฟล์ที่รองรับ: JPG, JPEG, PNG, WEBP, BMP, TIF, TIFF, HEIC
  • Resolution (รอบแรก): ย่อขนาดโดยให้ด้านที่ยาวที่สุดไม่เกิน 2048 px
  • Auto-Retry (รอบแก้ตัว): หาก AI แจ้งว่าอ่านเลขบัตรไม่ออก ระบบจะทำการอัปสเกลขอบเขตจำกัดเป็น 3072 px แล้วลองส่งใหม่อีกครั้ง
  • คุณภาพการบีบอัด: JPEG 85% (เป็นจุดสมดุลที่ดีที่สุดระหว่างขนาดไฟล์และความคมชัดของตัวหนังสือ)
  • อัตราส่วนภาพ: คงเดิม (Aspect Ratio) ไม่มีการตัดขอบ และไม่มีการหมุนภาพ

ตัวอย่าง: รูปถ่ายขนาด 4536×8064 พิกเซล จะถูกย่อเหลือ 1152×2048 (ในรอบแรก) หรือ 1728×3072 (ในรอบ Retry)

✅ ผลลัพธ์หลังการประมวลผล

เมื่อสิ้นสุดกระบวนการ ไฟล์ต้นฉบับจะถูกนำไปจัดเก็บที่ Output Directory โดยแบ่งออกเป็น 2 กรณี:

  • ผ่านการตรวจสอบ (Passed)

    ไฟล์จะถูกเปลี่ยนชื่อเป็น ชื่อ_เลขบัตร_ประเภทกู้.jpg แล้วย้ายเข้าไปเก็บในโฟลเดอร์ตามประเภทกู้ยืม เช่น Type1/, Type2/

  • ไม่ผ่านการตรวจสอบ (Failed)

    ย้ายไปเก็บที่โฟลเดอร์ Manual/ เพื่อให้เจ้าหน้าที่ตรวจด้วยตาเปล่า พร้อมระบุสาเหตุการตกหล่น เช่น ID mismatch (เลขบัตรไม่ตรง), Document is not signed (ไม่พบลายเซ็น)

⚙️ ค่าคอนฟิกหลักของระบบ (Configuration)

ผู้ดูแลระบบสามารถปรับแต่งพารามิเตอร์ต่างๆ ได้ที่ส่วนหัวของไฟล์ main.py:

GEMINI_MODEL_NAME = "gemini-2.5-flash"   # เปลี่ยนเป็น "gemini-2.5-pro" ได้หากต้องการความแม่นยำสูง
MAX_IMAGE_DIMENSION = 2048               # ขอบเขตความละเอียดรอบแรก
MAX_IMAGE_DIMENSION_RETRY = 3072         # ขอบเขตความละเอียดรอบ Retry
JPEG_QUALITY = 85                        # คุณภาพภาพ JPEG ที่จะส่งให้ AI
LOAN_TYPES = ["Type1", "Type2", "Type3", "Type4"] # กลุ่มประเภทการกู้ยืม
